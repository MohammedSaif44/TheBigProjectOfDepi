#🚗 Car Rental System API A production-ready Car Rental REST API built with ASP.NET Core 8, following Clean Architecture principles. The system supports full vehicle management, reservation booking, payment processing, notification delivery, reporting, and user authentication with JWT tokens. 📋 Table of Contents Project Overview Architecture Solution Structure Technologies Used Getting Started API Endpoints Data Models & Entities Design Patterns Used Database & Migrations 📌 Project Overview Car Rental System API is a comprehensive backend solution for an online car rental platform. It exposes RESTful endpoints covering: 🔐 Authentication — Register, login, role assignment, JWT token issuance 🚘 Cars — Vehicle listing, management, and image storage 📅 Reservations — Booking creation, tracking, and management 💳 Payments — Payment processing and tracking per reservation 🔔 Notifications — User notification delivery and preference management 📊 Reports — Rental activity and usage reporting 📧 Email — Transactional email sending via email templates The project uses Clean Architecture with 4 clearly separated layers: API, Application, Core, and Infrastructure. 🏗️ Architecture ┌─────────────────────────────────────────────┐ │ Car.Api │ │ (ASP.NET Core Web API) │ │ Controllers │ Program.cs │ appsettings │ └──────────────────┬──────────────────────────┘ │ calls ┌──────────────────▼──────────────────────────┐ │ Car.App │ │ (Application Layer) │ │ DTOs │ Interfaces │ Services │ │ AuthService · CarService · ReservationSvc │ │ PaymentService · NotificationService │ │ ReportService · EmailSender │ └──────────────────┬──────────────────────────┘ │ depends on ┌──────────────────▼──────────────────────────┐ │ Car.Core │ │ (Domain Layer) │ │ Entities: Car · Reservation · Payment │ │ Notification · NotificationPreference │ │ EmailTemplate · ApplicationUser │ └──────────────────┬──────────────────────────┘ │ implemented by ┌──────────────────▼──────────────────────────┐ │ Car.Infa │ │ (Infrastructure / Data Layer) │ │ ApplicationDbContext │ Migrations │ │ Repositories: Auth · Car · Reservation │ │ Payment · Notification · Report · Email │ └─────────────────────────────────────────────┘ Dependency Rule: Outer layers depend on inner layers. Service Layer Business logic is encapsulated in dedicated service classes in Car.App/Services/: AuthService — registration, login, JWT generation, role assignment CarService — vehicle CRUD, image file handling ReservationService — booking creation, status management, session tracking PaymentService — payment processing and status updates NotificationService — notification creation and delivery ReportService — aggregation queries for reports EmailSender — SMTP email dispatch using EmailTemplate records Dependency Injection All interfaces and their implementations are registered in Program.cs. Controllers and services receive dependencies via constructor injection — no new keyword for services anywhere. 🗄️ Database & Migrations Single SQL Server database managed by ApplicationDbContext. MigrationDescription20251023183031_InitInitial schema — Cars, Reservations, Payments, Users20251023183258_FixDecimalPrecisionFixed decimal precision on price/amount columns20251024114647_SeedAdminSeeds the default admin user account20251119105258_AddSessionIdAdded SessionId column to Reservations20251119173938_AddNotificationAdded Notifications and NotificationPreferences tables20251121155940_AddCarImageAdded ImageUrl column to Cars 📝 Notes Car images are stored locally in wwwroot/images/cars/ using GUID filenames to avoid conflicts. For production, consider migrating to cloud storage (Azure Blob, AWS S3). The SeedAdmin migration automatically provisions an admin account on first run — update the seeded credentials before deploying to production. The SessionId on reservations supports tracking multi-step booking flows. All protected endpoints require a valid JWT Bearer token passed as Authorization: Bearer <token>. Email templates are stored in the database (EmailTemplate entity), allowing admins to update email content without redeploying.
